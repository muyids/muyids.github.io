# 分布式文件系统

# 场景分析

需求1：用户读文件，写文件

文件空间有多大？ > 1000T

需求2: 多台机器存储这些文件

支持机器数量 > 10万台

# Service 服务

Client + Server

Server架构

Peer to Peer (社会主义)

优点：一台机器挂了还可以工作；缺点：多台机器需要经常通信保持数据一致

Master Slave (资本主义，一个老大哥)

优点：设计简单，数据容易保持一致；缺点：单master不可靠
